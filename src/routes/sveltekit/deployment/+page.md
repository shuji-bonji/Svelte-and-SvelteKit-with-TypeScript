## デプロイ戦略

SvelteKitは様々なデプロイ戦略をサポートしています：

### 静的サイト生成（SSG）
```bash
npm run build
# 静的ファイルが build/ に生成される
```

### サーバーサイドレンダリング（SSR）
```bash
npm run build
node build/index.js
# Node.jsサーバーとして実行
```

## プラットフォーム比較

各プラットフォームの特徴：

:::info[プラットフォーム選択の指針]
- **Vercel**: SvelteKit公式推奨、簡単デプロイ、Edge Functions
- **Netlify**: 静的サイト中心、豊富なプラグイン、Forms機能
- **Cloudflare Pages**: 高性能CDN、Workers統合、低価格
- **AWS**: 柔軟性、スケーラビリティ、コスト管理が重要
- **自前サーバー**: 完全制御、カスタマイズ性、運用責任
:::

:::tip[アダプター選択]
SvelteKitのアダプターシステムにより、同一のコードベースを異なるプラットフォームに最適化してデプロイできます。開発段階でプラットフォームを決める必要はありません。
:::

## セキュリティ考慮事項

本番環境では以下のセキュリティ対策が重要です：

- **HTTPS必須**: すべての通信をHTTPS化
- **CSP設定**: Content Security Policyによる攻撃防御
- **環境変数**: 機密情報の安全な管理
- **依存関係**: 脆弱性スキャンと更新
- **アクセス制御**: 適切な認証・認可

## トラブルシューティング

本番環境でよくある問題と解決方法：

- **ビルドエラー**: 依存関係とNode.jsバージョンの確認
- **404エラー**: ルーティング設定とアダプター設定の確認
- **パフォーマンス問題**: 監視ツールでのボトルネック特定
- **セキュリティ**: 脆弱性スキャンと設定レビュー
